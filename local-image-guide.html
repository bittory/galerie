저작권 분석 — 로컬 저장 가능 여부

소스라이선스로컬 저장상업 배포출처 표기
Met Museum CC0 (Public Domain) ✅ 가능 ✅ 가능 권장
Wikimedia Commons Public Domain / PD-old ✅ 가능 ✅ 가능 권장
국립중앙박물관 공공누리 제1유형 ✅ 가능 ✅ 가능 법적 의무
이중섭 작품 사후 70년 PD (한국법) ✅ 가능 ✅ 가능 권장
✅ 결론: 전체 111개 작품 로컬 저장 및 상업 배포 합법
CC0와 Public Domain은 세계 어디서든 어떤 용도로도 사용 가능합니다. 국립중앙박물관(공공누리 1유형)은 출처 표기 의무가 있으며, 현재 설정 페이지 크레딧 섹션에 이미 표기되어 있어 법적 요건을 충족합니다.

출처 표기 해결 방안 (A + B 동시 적용 권장)

방안 A — 설정 페이지 크레딧 (현재 구현됨)
설정 → 저작권 크레딧 섹션에 이미 모든 출처 표기 중.
국립중앙박물관 공공누리 1유형 의무 충족. 추가 작업 불필요.
방안 B — 앱 최초 실행 시 1회 출처 Notice (추가 구현)
앱 첫 설치 후 1회만 "이 앱은 CC0/PD 공개도메인 명화를 사용합니다" 화면 표시.
앱스토어 심사 통과율 향상 + 사용자 신뢰도 상승 효과. 다음 세션에서 구현 예정.

최종 프로젝트 폴더 구조

galerie-app/ www/ index.html ← 메인 앱 (URL 로컬로 교체됨) galerie-cast-receiver.html images/ full/ 001_별이_빛나는_밤_full.jpg ← 배경용 (1280px) 002_해바라기_full.jpg ... 111개 thumb/ 001_별이_빛나는_밤_thumb.jpg ← 갤러리 썸네일 (400px) 002_해바라기_thumb.jpg ... 111개 android/ capacitor.config.json

실행 순서

1
Python 설치 확인
python --version # 3.6 이상이면 OK # 없으면: python.org → Download Python 3.12
2
갤러리 앱 폴더에 스크립트 배치
download_artworks.py와 replace_urls.py를
galerie-app 루트 (www 폴더와 같은 위치)에 복사
3
이미지 일괄 다운로드 실행
cd galerie-app python download_artworks.py # 진행 상황 출력 예시: [ 1/111] 별이 빛나는 밤 (Wikimedia PD) ✅ full 450KB ✅ thumb 62KB [ 2/111] 해바라기 (Wikimedia PD) ✅ full 380KB ✅ thumb 58KB ... 완료: full 111/111 · thumb 111/111
소요 시간: 약 5~10분 (네트워크 속도에 따라)
실패한 항목은 failed_list.txt에 저장됨 → 재실행하면 이어서 다운로드
4
HTML URL 로컬 경로로 교체
python replace_urls.py # 결과: ✅ 백업 생성: www/index.html.bak full 교체: 111개 thumb 교체: 111개 저장 완료: www/index.html
5
APK 빌드 전 동기화
npx cap sync android # www/ 전체(이미지 포함)를 android/에 복사 npx cap run android # Fold 7 설치 및 실행
이미지가 앱 내부에 번들링되어 오프라인에서도 동작합니다.
6
GitHub 업로드 (웹 버전용)
GitHub에도 images 폴더를 함께 올리면 GitHub Pages에서도 이미지가 로컬 로드됩니다.
# git으로 올리는 경우 git add www/images/ www/index.html git commit -m "feat: 이미지 로컬 저장으로 전환" git push
⚠ images 폴더 용량이 ~50MB이므로 git push 시간이 다소 소요됩니다. LFS 설정 불필요 (파일당 100MB 이하).
